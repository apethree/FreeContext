This is a quick preview for teams planning to build around OneShot from day one.

## What is ready for developers

- Cloud architecture docs covering sessions, sync, and channel handling.
- Webhook documentation for inbound and outbound automation patterns.
- Channel integration documentation for operational workflows.

## Suggested first integration path

If you are evaluating OneShot for internal automation, start in this order:

1. Read [Cloud Overview](/cloud/overview/).
2. Configure [Webhooks](/cloud/webhooks/).
3. Connect one channel and validate end-to-end message delivery.

## What we will publish at public launch

- Public release notes and binary availability updates.
- Expanded examples for webhook payload validation and retry behavior.
- Operational guidance for production rollouts.

If you are preparing now, the docs are stable enough to plan your first proof of concept.
